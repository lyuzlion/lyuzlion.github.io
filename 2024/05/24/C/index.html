地址空间划分

系统运行程序时，为其所分配的内存一般包括如下几个部分：

  1. 代码区（存放代码段，里面是一些二进制机器指令）
  2. 常量区（const、字符串常量）
  3. 全局数据区（全局变量、static）
  4. 堆区（malloc）
  5. 栈区（局部变量、函数参数）
  6. 命令行参数区（存主函数main的参数）

注意：寄存器的值不保存在地址空间中。保存在PCB中。

全局数据区再分为

  1. 已初始化的数据区
  2. 未初始化的数据区（BSS，Block Started by Symbol，是指存放程序中未初始化的全局变量和静态变量的一块内存区域。特点是:可读写的，在程序执行之前BSS段会自动清0）

x86中的堆、栈:

  1. 堆：向上生长；(低地址高地址)
  2. 栈：向下生长；(高地址低地址)

逻辑空间：

  1. 代码段或文本段(code，.text)：存放可执行指令
  2. 数据段(data)：存放程序的全局变量与静态局部变量
    1. .rodata(只读数据段)：字符串常量、全局const变量等常量，程序运行时不可修改
    2. .rwdata(读写数据段)：程序运行时可以修改全局变量
    3. .zidata/.bss(零初始化数据段)：未初始化/初始化为0的全局变量
  3. 堆(.heap)：运行时动态分配的内存（malloc，calloc，realloc）
  4. 栈(.stack)：包含临时数据，如局部变量、函数参数、返回地址

栈底一般是main函数的入口，在将.bss段初始化0等初始化工作后，开始执行main函数

open()

头文件：

函数原型：

flags:

  1. O_RDONLY 只读
  2. O_WRONLY 只写
  3. O_RDWR 可读写
  4. O_CREAT 若欲打开的文件不存在则自动建立该文件，与上面三个组合使用，例如O_CREAT|O_RDWR

进程A第一次打开文件x，会在文件表中创建一个引用数为1的引用，再次打开文件x，再创建一个新的引用（而不是原有引用数加一）

关于父子进程打开文件的问题，如果先创建子进程，父子进程再打开相同的文件，那么文件表里的引用其实是相互独立的；但是如果父进程先打开文件再创建子进程，子进程继承父进程的资源，会导致原先的引用数变成2。

用户文件描述符表（也叫局部的文件打开表）、文件表（也叫全局的文件打开表）、索引节点表（也叫全局的FCB表）都由操作系统内核来维护，保存在OS内核区，也就是在内存里。

系统的索引节点表:

open的过程：

  1. 解析open系统调用给出的文件路径，确定文件名所在的文件目录表，并把它装入内存；
  2. 根据文件名查找文件目录表，找到该文件所在的目录项（也就是找到索引节点号）；如果不存在文件，返回错误（不存在该文件）
  3. 根据目录项，在磁盘上找到要打开文件的FCB，读入到内存（更精确地，OS内核里）的索引节点表中（变成Active FCB），并设置相关信息
    在全局文件表中为该文件分配一个表项，建立指向索引节点表中对应的FCB项的指针，并设置其它相关信息
  4. 在进程的文件描述符表中（U-area，PCB的一部分）建立指向全局文件表中的相应项的指针
  5. Open() 返回该文件在文件描述符表中的偏移，即File Descriptor；

一些等价的说法：

  1. 进程的文件描述符表局部的打开文件表
  2. 全局文件表文件表系统级打开文件表
  3. 文件目录表目录表
  4. 打开文件句柄全局的打开文件表项

fread()

头文件：


函数原型：

  1. void *buffer 参数 : 将文件中的二进制数据读取到该缓冲区中 ;
  2. size_t size 参数 : 读取的 基本单元 字节大小, 单位是字节, 一般是 buffer 缓冲的单位大小;如果 buffer 缓冲区是 char 数组 , 则该参数的值是 sizeof(char);
    如果 buffer 缓冲区是 int 数组 , 则该参数的值是 sizeof(int);
  3. size_t count 参数 : 读取的基本单元个数;
  4. FILE *stream 参数 : 文件指针;
  5. size_t 返回值 : 实际从文件中读取的基本单元个数 ; 读取的字节数是基本单元数 * 基本单元字节大小;在文本文件中，一个字符就是一个字节。

注意：

  1. 在Windows下，每行末尾是回车换行，\r\n
  2. 在Unix下，每行末尾是换行，\n
  3. 在Windows下，如果 fopen是”r”文本方式打开, 读文件时会自动转换换物符\r\n为 \n。
  4. 在Windows下，如果 fopen是”r”文本方式打开, 写文件时会自动转换换物符\n为 \r\n。
  5. 在Windows下，如果 fopen是”rb”是以二进制方式打开，不会自动转换。

应用实例：一次性读满整个缓冲区

如果缓冲区很小，需要多次读入文件数据。此时使用feof(p)判断文件是否已经读到文件结束符EOF（而不是文件最后一个字符）。
